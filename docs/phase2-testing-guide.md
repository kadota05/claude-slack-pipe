# Phase 2 テストガイド

## Phase 1 からの追加設定

### Slack App 設定（api.slack.com）

#### 追加が必要な OAuth Scopes（Bot Token Scopes）

| Scope | 用途 | Phase 1 | Phase 2 |
|-------|------|---------|---------|
| `reactions:read` | 🔴 interrupt リアクション検知 | 不要 | **必要** |
| `reactions:write` | ⏳🧠✅ リアクション追加/削除 | あり | あり |
| `chat:write` | メッセージ投稿 | あり | あり |
| `im:history` | DM メッセージ読み取り | あり | あり |
| `im:write` | DM 送信 | あり | あり |

#### 追加が必要な Event Subscriptions

| Event | 用途 | Phase 1 | Phase 2 |
|-------|------|---------|---------|
| `reaction_added` | 🔴 リアクション検知 → interrupt 送信 | 不要 | **必要** |
| `message.im` | DM メッセージ受信 | あり | あり |
| `app_home_opened` | Home Tab 表示 | あり | あり |

#### Interactivity（Block Kit アクション）

Phase 2 では Home Tab に `static_select` と `button` を使用するため、Slack App の **Interactivity** が有効になっている必要がある（Socket Mode 使用時は自動的に有効）。

### 環境変数（.env）

```bash
# Phase 2 で追加
DATA_DIR=~/.claude-slack-pipe/    # 永続ストアのデータディレクトリ（デフォルト: ~/.claude-slack-pipe/）
```

### Claude CLI 要件

Phase 2 では以下のフラグを使用するため、Claude CLI の最新版が必要：

```
--input-format stream-json    # 永続プロセスの stdin
--output-format stream-json   # ストリーミング出力
--replay-user-messages        # セッション復元時のメッセージリプレイ
--verbose                     # 詳細なイベント出力
```

**確認コマンド:**
```bash
claude --version
# 0.2.x 以上が必要
```
