# ボディコンシェルジュ設計書

## 概要

専属ボディコンシェルジュとして、食事・運動・体調・美容を統合的にマネジメントするAIアシスタント。専用Slackチャネルを通じて双方向にやり取りし、使えば使うほどパーソナライズされていく。

## ペルソナ

**専属ボディコンシェルジュ**

- 栄養士の知識をコアに、トレーナーの視点で身体づくりを伴走
- 健康だけでなく「どう見えるか・どう在りたいか」（美意識）も守備範囲
- 「タンパク質不足です」ではなく「タンパク質不足は肌のハリに出ますよ」
- プロだけど温かい。褒める。責めない
- 「ダメ」ではなく「こうするともっと良い」
- 体調が悪いときは医師の受診を勧める（診断はしない）

## アーキテクチャ

### 独立プロジェクト型（B型）

claude-slack-pipeとは独立したプロジェクトとして構築する。

```
~/dev/body-concierge/        ← 独立プロジェクト
├── src/
│   ├── process-message.ts   # ユーザーメッセージ処理エントリポイント
│   ├── scheduled/
│   │   ├── morning.ts       # 朝のコメント生成（07:00）
│   │   ├── evening.ts       # 夜の振り返り+knowledge更新（22:00）
│   │   └── weekly.ts        # 週間レポート+深いknowledge更新（日曜20:00）
│   ├── prompts/             # プロンプトテンプレート
│   ├── slack.ts             # Slack API投稿ユーティリティ
│   └── data.ts              # データ読み書きユーティリティ
├── data/
│   ├── profile.json         # ユーザー基本情報
│   ├── meals/               # 日別食事ログ
│   ├── metrics/             # 体重・体調・睡眠等
│   ├── interactions/        # 相談・質問のやり取りログ
│   ├── knowledge.md         # パーソナライズ知識（AI自律更新）
│   └── weekly/              # 週間レポート履歴
├── CLAUDE.md                # コンシェルジュの人格・ルール
├── docs/
└── package.json
```

### Bridge側変更: Channel Router抽象化

BridgeにDM/チャネルの分類層とチャネルルーティング層を追加する。

```
メッセージ着信
  ↓
[Event Classification Layer]
  channel_type === 'im' → 既存DM処理（変更なし）
  channel_type === 'channel' or 'group' → Channel Router
  ↓
[Channel Router]
  slack-memory.json で channelId を検索
  → 見つかった → そのプロジェクトの handler を実行
  → 見つからない → 無視
```

**DM処理のコードパスには一切触れない。** 既存機能への影響ゼロ。

#### slack-memory.json（ルーティングテーブル）

場所: `~/.claude-slack-pipe/slack-memory.json`

```json
[
  {
    "folder": "~/dev/body-concierge",
    "description": "ボディコンシェルジュ",
    "channel": "#body-concierge",
    "channelId": "C0123...",
    "handler": "src/process-message.ts",
    "createdAt": "2026-03-27"
  }
]
```

- Bridge起動時にロード
- fs.watchでファイル変更を監視し、再読み込み
- ファイルが存在しなければチャネルルーティング無効（DMのみ動作）
- 新しいチャネルプロジェクト追加はこのファイルに1行足すだけ

#### 標準ハンドラーインターフェース

Bridgeが外部プロジェクトのハンドラーを呼び出す際の標準形式:

```bash
tsx {folder}/{handler} \
  --text "テキスト" \
  --files '/tmp/img1.jpg,/tmp/img2.png' \
  --user-id "U..." \
  --channel-id "C..." \
  --thread-ts "1711..." \
  --timestamp "1711..."
```

- ハンドラー自身がSlack APIで応答を投稿する（Bot Tokenはclaude-slack-pipeと同じSlack Appのものを使用。ハンドラープロジェクトの.envに設定）
- Bridgeはファイルのダウンロード（Slack API経由）と一時ファイルへの保存を行い、そのパスをハンドラーに渡す
- Bridgeの役割はメッセージ+ファイルの受け渡しのみ、応答には関与しない

## データモデル

### profile.json

初回セットアップ時に作成。ユーザーが目標変更した際に更新。

```json
{
  "age": 30,
  "height_cm": 175,
  "weight_kg": 70,
  "goals": ["減量", "肌改善"],
  "allergies": [],
  "medications": [],
  "exercise_habits": "週3ジム",
  "updated_at": "2026-03-27"
}
```

### meals/YYYY-MM-DD.json

```json
{
  "date": "2026-03-27",
  "meals": [
    {
      "time": "12:30",
      "type": "lunch",
      "description": "チキンサラダ",
      "image": true,
      "nutrition": {
        "calories": 350,
        "protein_g": 25,
        "fat_g": 15,
        "carbs_g": 20
      },
      "source": "photo_analysis"
    }
  ]
}
```

### metrics/YYYY-MM-DD.json

```json
{
  "date": "2026-03-27",
  "weight_kg": 69.5,
  "sleep": "6h、浅め",
  "condition": "やや疲れ",
  "skin": null,
  "exercise": "ジム 胸・肩 45分",
  "water_ml": 1500
}
```

### interactions/YYYY-MM-DD.json

相談・質問など構造化データ以外のやり取りを記録。

```json
{
  "date": "2026-03-27",
  "entries": [
    {
      "time": "14:00",
      "type": "question",
      "input": "最近肌荒れひどい",
      "response": "直近1週間の食事を見ると..."
    },
    {
      "time": "18:00",
      "type": "goal_change",
      "input": "肌改善を優先したい",
      "response": "目標を更新しました"
    }
  ]
}
```

### knowledge.md

AIが自律的に読み書きするパーソナライズ知識ファイル。人間にも読める形式。

```markdown
# パーソナライズ知識

## 食の好み
- [確信度:高] 和食好き、特に魚
- [確信度:中] 辛いものは避ける傾向
- [確信度:低] パスタは月1程度

## 身体の傾向
- [確信度:高] 乳製品多い週→肌荒れ
- [確信度:中] 睡眠6h以下の翌日は食事が乱れる

## 生活パターン
- [確信度:高] 金曜は外食/飲み会
- [確信度:高] 月曜朝は食欲低め
- [確信度:中] ジムは火・木・土

## 提案への反応
- サラダ系→よく採用
- 魚メニュー→3回連続スルー（苦手ではなく面倒？）
- 朝プロテイン+バナナ→定着済み

## 目標の変遷
- 3/27: 減量+肌改善で開始

## 最終更新: 2026-03-27
```

#### 確信度の昇格ルール

- 低 → 中: 同じ傾向を2週連続で観測
- 中 → 高: 4週以上一貫して観測
- 高 → 削除: 明確に覆るデータあり

#### サイズ管理

- 最大目安: 200行
- 週間ジョブで整理・統合
- 古い/覆された情報は削除

## 利用シーン

### ユーザー発信: 記録系

サクッと送る→サクッと返る。アドバイスは定期コメントに回す。

| 送る内容 | コンシェルジュの反応 |
|---------|-----------------|
| 食事（写真/テキスト） | メニュー認識+栄養推定 |
| 間食・飲み物 | 同上 |
| 体重 | 記録+推移の一言 |
| 運動した内容 | 記録 |
| 睡眠の質 | 記録 |
| 水分摂取 | 記録 |
| 体調（だるい等） | 記録 |
| 肌の状態 | 記録 |

### ユーザー発信: 相談系

聞かれたらしっかり答える。データ+knowledge注入で個別化された回答。

| 相談内容 | 関わり方 |
|---------|---------|
| 「○○食べたい」 | 目標との兼ね合いで提案 |
| 外食・飲み会の予定 | 事前アドバイス+翌日ケア |
| 「最近どう？」傾向確認 | データに基づく分析 |
| イベント準備（デート等） | 逆算した食事・運動プラン |
| サプリ相談 | 食事データ踏まえて提案 |
| 目標変更 | プロフィール更新+方針調整 |

### コンシェルジュ発信: 定期コメント

| タイミング | 内容 |
|-----------|------|
| 毎朝 07:00 | 昨日の状況を踏まえた今日のフォーカス |
| 毎晩 22:00 | 1日の振り返り（栄養サマリ+体調相関+よかった点）+ knowledge更新 |
| 週1 日曜20:00 | 週間レポート（体重推移・栄養傾向・肌/体調変化・来週の提案）+ 深いknowledge更新 |

気づき（パターン検出）は全て毎晩・週1のコメントにまとめて届ける。リアルタイムでは出さない。

## メッセージ処理フロー

### process-message.ts

```
1. 入力を分類（claude -p haiku）
   → meal / snack / weight / exercise
     / sleep / condition / skin / water
     / question / goal_change

2. 分類に応じた処理（claude -p sonnet）
   → meal: 写真+テキストから栄養推定
   → weight: 数値抽出
   → question: データ+knowledge読み込み+回答生成
   → etc.

3. 結果をデータファイルに保存

4. 応答をSlack APIで投稿
```

### モデル使い分け

| 処理 | モデル | 理由 |
|------|--------|------|
| 分類 | haiku | 高速・低コスト |
| 栄養推定 | sonnet | 画像認識+推定 |
| 相談回答 | sonnet | バランス良い |
| 定期コメント | sonnet | コスト重視 |
| knowledge更新 | 夜/週間ジョブにバンドル | 追加コストなし |

## パーソナライズエンジン

### knowledge更新タイミング

毎メッセージごとには更新しない。定期ジョブにバンドルしてコスト・遅延を抑える。

| タイミング | 読むデータ | 更新の深さ |
|-----------|-----------|-----------|
| 毎晩22:00 | 当日のmeals + metrics + interactions + knowledge.md | 新しい好み・傾向の追加、確信度微調整 |
| 週1日曜20:00 | 過去7日の全データ + knowledge.md + 前回weeklyレポート | 深いパターン認識、確信度昇格/降格、矛盾解消、古い情報の整理 |

### knowledge利用タイミング

| 処理 | knowledge参照 |
|------|-------------|
| 食事記録 | しない（軽量に） |
| 相談回答 | する（個別化） |
| 朝コメント | する（提案の質を上げる） |
| 夜振り返り | する（更新のため） |
| 週間レポート | する（深い分析のため） |

## 初回セットアップフロー

1. プロジェクト作成（~/dev/body-concierge/）
2. Slackチャネル作成（#body-concierge）
3. slack-memory.jsonにルーティング登録
4. Bridge再起動（Channel Router有効化）
5. チャネルに初回メッセージを投稿してプロフィール収集
6. launchdジョブ登録（朝・夜・週間の3つ）

### 初回メッセージ

コンシェルジュが最初の挨拶を投稿:

> はじめまして、あなた専属のボディコンシェルジュです。
>
> まずあなたのことを教えてください：
> ・年齢
> ・身長・体重
> ・目標（減量/増量/維持/肌改善 等）
> ・アレルギーや持病があれば
> ・運動習慣
>
> テキストで自由に送ってもらえれば、こちらで整理します。

ユーザーの返答をclaude -pで解析してprofile.json作成、確認メッセージを投稿。
