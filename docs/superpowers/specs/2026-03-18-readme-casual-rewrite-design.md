## セクション構成

### 1. これは何？

- このプログラムを動かすとSlackのDMから自分のPCのClaude Code CLIを直接操作できる
- スマホから操作できるのが革命的で、PCが開けないときに便利
- 旅行、散歩、満員電車など、PCが使えないあらゆる外出先で使える

### 2. セットアップ

- Claude Codeでこのプロジェクトを開き、「**セットアップして**」と言うだけ
- 対話形式でインストールと設定ファイルの生成が完了する
- 手動手順は書かない

### 3. 使い方

以下の体験フローに沿って説明する:

1. **ホームタブで準備** — モデル（Opus / Sonnet / Haiku）と作業ディレクトリをドロップダウンから選ぶ（Claude Codeのプロジェクト一覧から自動取得される）
2. **メッセージタブでプロンプトを送る**
3. **リアクションで進捗がわかる**:
   - ⏳ = SlackのメッセージをローカルのNode.js（親プロセス）が受け取って、Claude（子プロセス）にプロンプトを渡すまでの準備時間。「システムが受け取ってAIに渡す準備をしている」という意味
   - 🧠 = 子プロセスのClaudeがプロンプトを受け取って処理を実行中。「AIが処理しています」という意味
   - ✅ = 処理完了
4. **処理を中断したいとき** — 🧠がついている自分のメッセージに🔴リアクション（「あか」で変換）をつけると処理がストップする
5. **スレッドとコンテキスト** — AIは人間が立ち上げたスレッドに返信する形で回答する。コンテキストはスレッド内で完結し、スレッド間では分断される。例えば「さっきのスレッドで言ったことについて検討して」と言ってもAIは理解できない

### 4. 注意事項

- ファイル・画像の添付は未対応。必ずテキストだけ送ること
- Permission modeは最強設定（常にbypass permissions onのイメージ）。仕様上変更不可なので注意
- Slack上で `/なんたら` とスラッシュコマンドを打つとSlackが用意したコマンドしか使えずエラーになる。カスタムコマンドは検討中で未対応。スキルを使いたい場合は「superpowersのスキルを使って」などテキストで指示すること

### 6. 仕組み

Mermaidダイアグラムで概念図を描く。説明の粒度は以下の程度:

- SlackとローカルPCで立ち上げているNode.jsプロセス（親プロセス）がSocket Modeで繋がる
- Node.jsはSlackから来た情報を子プロセスのClaudeに流すために整形して、`claude -p`（stdin/stdoutでClaudeを使えるコマンド）でstdinで渡す
- 結果が親プロセスのNode.jsへstdoutで返ってくるので、それをSlackで表示したい形にしてAPIで返す

技術詳細（stream-json形式、JSON構造など）はダイアグラムのラベルに自然に含める程度で、本文では説明しない。

### Mermaidダイアグラムの要素

```
Slack Cloud ←―Socket Mode(WebSocket)―→ Node.js（親プロセス、ローカルPC）
                                            ↕ stdin/stdout
                                        Claude CLI（子プロセス）
```

ラベルに含める情報:
- Slack Cloud側: 「DM送信」「結果表示」
- Node.js側: 「メッセージ整形」「結果をSlack形式に変換」
- 接続方式: Socket Mode（上り）、Slack API（下り）
- CLI側: `claude -p`、stdin/stdout
