# Phase 2 実機テストシナリオ

各テストは独立して実行可能。結果を `[ ]` → `[x]` / `[!]` で記録する。

---

## 1. アンカーメッセージ

### 1-1. 新規セッション開始時にアンカーが投稿される
- [ ] ボットにDMで「hello」と送信
- [ ] スレッド内にアンカーメッセージが投稿される
- [ ] アンカーにヘッダー（セッション名）が表示される
- [ ] ステータス `:large_green_circle: Active Session` が表示される
- [ ] プロジェクトパスが表示される
- [ ] Session ID、開始時刻、コスト `$0.00` が表示される
- [ ] モデル選択 `static_select` (Opus/Sonnet/Haiku) が表示される
- [ ] 「▲ Collapse」ボタンが表示される
- [ ] 「Commands」「End Session」ボタンが表示される
- [ ] フッターに「Send a message in this thread...」が表示される

### 1-2. 同じスレッドで2回目のメッセージ → アンカーは再投稿されない
- [ ] 同じスレッドに「もう1つ質問」と送信
- [ ] 新しいアンカーは投稿されない（1つ目のアンカーのみ）
- [ ] 応答完了後、既存アンカーのコスト/トークンが更新される

---

## 2. アンカー折りたたみ / 展開

### 2-1. ボタンで折りたたみ
- [ ] アンカーの「▲ Collapse」ボタンをクリック
- [ ] アンカーが1行表示に変わる（`:large_green_circle: *セッション名* | Model | N% | $X.XX`）
- [ ] 「▼ Expand」ボタンが表示される

### 2-2. ボタンで展開
- [ ] 折りたたまれたアンカーの「▼ Expand」をクリック
- [ ] フル表示に戻る（全ブロック表示）

### 2-3. テキストコマンドでトグル
- [ ] スレッド内で `cc /panel` と送信
- [ ] アンカーが折りたたまれる（or 展開される）
- [ ] もう一度 `cc /panel` → 元に戻る

---

## 3. モデル選択

### 3-1. アンカーのselectからモデル変更
- [ ] アンカーの `static_select` で「Haiku」を選択
- [ ] アンカーが更新され、selectの選択値が「Haiku」になる
- [ ] 次のメッセージ送信時にHaikuモデルが使用される（応答速度・品質で判断）

### 3-2. テキストコマンドでモデル変更
- [ ] `cc /model opus` と送信
- [ ] アンカーが更新され、selectが「Opus」に変わる
- [ ] 通知メッセージ「Model changed to opus」が表示される

### 3-3. 無効なモデル名
- [ ] `cc /model gpt4` と送信
- [ ] エラーメッセージ「Usage: cc /model <opus|sonnet|haiku>」が表示される
- [ ] アンカーは変更されない

---

## 4. コマンドモーダル

### 4-1. モーダルを開く
- [ ] アンカーの「Commands」ボタンをクリック
- [ ] モーダルが開く
- [ ] タイトルが「コマンド一覧」
- [ ] 4カテゴリ表示: セッション管理 / 開発 / モデル設定 / その他
- [ ] 各カテゴリに `static_select` がある
- [ ] 引数入力欄がある
- [ ] 「実行」「キャンセル」ボタンがある

### 4-2. コマンド実行（引数なし）
- [ ] モーダルで「セッション管理」→ `/status` を選択
- [ ] 「実行」をクリック
- [ ] スレッドにステータス情報が投稿される

### 4-3. コマンド実行（引数あり）
- [ ] モーダルで「セッション管理」→ `/rename` を選択
- [ ] 引数欄に「テスト用セッション」と入力
- [ ] 「実行」をクリック
- [ ] アンカーのヘッダーが「テスト用セッション」に更新される

### 4-4. キャンセル
- [ ] モーダルで「キャンセル」をクリック
- [ ] モーダルが閉じ、何も実行されない

---

## 5. セッション終了

### 5-1. ボタンで終了
- [ ] アンカーの「End Session」をクリック
- [ ] 確認ダイアログ「End this session?」が表示される
- [ ] 「End」をクリック
- [ ] アンカーのステータスが `:white_circle: Session Ended` に変わる
- [ ] ボタン/selectはそのまま残る（再操作は無効化される）

### 5-2. テキストコマンドで終了
- [ ] 新しいスレッドでセッション開始後、`cc /end` を送信
- [ ] 終了メッセージ「Session ... ended.」が表示される
- [ ] アンカーが終了状態に更新される

### 5-3. 終了済みセッションで操作
- [ ] 終了済みスレッドでモデルselectを変更しようとする
- [ ] 何も起こらない（エラーなし）

---

## 6. ストリーミング進捗表示 (Message A)

### 6-1. 基本的な進捗表示
- [ ] 新しいスレッドで「src/index.tsの内容を要約して」と送信
- [ ] Message A（進捗メッセージ）がスレッドに投稿される
- [ ] `⏳ 処理中... (ステップ 0/0) | 経過: 0s` が最初に表示される
- [ ] ツール使用が始まると、ステップ数が増える
- [ ] `🔄 Read ...` → `✅ Read src/index.ts (N行)` のように更新される

### 6-2. 複数ツール使用の進捗
- [ ] 「src/slack/配下のファイルを全部読んで概要をまとめて」と送信
- [ ] 複数の `✅ Read ...` 行が順次表示される
- [ ] ステップカウントが正しく増加する
- [ ] 経過時間が更新される

### 6-3. 思考スニペット表示
- [ ] 処理中に `💭 _...テキスト..._` が表示されるか確認
- [ ] （表示されるかはClaudeの応答次第）

### 6-4. 完了表示
- [ ] 処理完了後、Message Aが `✅ 完了 | Nステップ | Ns | $X.XXX` に変わる
- [ ] 「📋 全ログを見る」ボタンが表示される

---

## 7. 結果表示 (Message B)

### 7-1. 通常の結果
- [ ] 処理完了後、Message B（結果テキスト）が別メッセージとして投稿される
- [ ] Message AとMessage Bが両方スレッドに存在する

### 7-2. リアクション
- [ ] 送信したメッセージに ⏳ リアクションが付く
- [ ] 完了後 ✅ に変わる

### 7-3. エラー時
- [ ] 「存在しないファイル /nonexistent/path/xxx.ts を編集して」と送信
- [ ] エラーが発生した場合、❌ リアクションに変わる
- [ ] エラーメッセージがBlock Kit形式で表示される

---

## 8. 長文応答 (files.uploadV2)

### 8-1. 分割表示（3,900〜39,000文字）
- [ ] 長い応答が返るプロンプトを送信（例: 「package.jsonの全依存パッケージについて詳しく説明して」）
- [ ] 3,900文字を超えた場合、複数メッセージに分割される
- [ ] コードブロックの途中で分割されない

### 8-2. ファイルアップロード（39,000文字超）
- [ ] 非常に長い応答が返るプロンプトを送信（発生しにくいため、テスト困難な場合はスキップ可）
- [ ] 39,000文字超の場合、`.md` ファイルとしてアップロードされる
- [ ] 「Response was too long to display inline.」コメントが付く

## 9. 詳細モーダル

### 9-1. 進捗詳細モーダル

- [ ] モーダルが開き、各ステップの詳細が表示される

### 9-2. 全ログモーダル

## 10. セッション命名

### 10-1. 自動命名
- [ ] 「認証機能を実装して」と送信してセッション開始
- [ ] アンカーのヘッダーが `project: 認証機能を実装して` のような形式になる

### 10-2. 手動命名（テキストコマンド）
- [ ] `cc /rename 新しい名前` と送信
- [ ] アンカーのヘッダーが「新しい名前」に更新される
- [ ] 通知メッセージが表示される

### 10-3. 長い名前の切り詰め
- [ ] `cc /rename ` + 200文字の名前を送信
- [ ] 150文字で切り詰められる

## 11. Home Tab

### 11-1. 基本表示
- [ ] Slack左パネルでボットアプリを選択 → 「Home」タブを開く
- [ ] ヘッダー「Claude Code Bridge」が表示される
- [ ] `:large_green_circle: Bridge Running` が表示される
- [ ] プロジェクト一覧が表示される

### 11-2. アクティブセッション表示
- [ ] アクティブなセッションがある状態でHome Tabを開く
- [ ] 「Active Sessions」セクションにセッションが表示される
- [ ] 「Open Thread」ボタンがある

### 11-3. 終了済みセッション履歴
- [ ] セッションを `cc /end` で終了してからHome Tabを再読み込み
- [ ] 「最近のセッション（終了済み）」セクションが表示される
- [ ] 終了したセッション名と経過時間が表示される

## 12. ブリッジコマンドのアンカー連携

### 12-1. cc /model
- [ ] `cc /model haiku` と送信
- [ ] アンカーのモデルselectが「Haiku」に更新される

### 12-2. cc /rename
- [ ] `cc /rename テスト名` と送信
- [ ] アンカーのヘッダーが「テスト名」に更新される

### 12-3. cc /panel
- [ ] `cc /panel` と送信
- [ ] アンカーが折りたたまれる（テキスト応答なし、直接chat.update）

### 12-4. cc /status
- [ ] `cc /status` と送信
- [ ] セッション情報（ID, Project, Model, Turns, Cost, Tokens）が表示される

### 12-5. cc /help
- [ ] `cc /help` と送信
- [ ] コマンド一覧が表示される

## 13. エッジケース

### 13-1. 同時実行制限
- [ ] 1つのプロンプトが処理中に、同じスレッドで別のメッセージを送信
- [ ] 「You already have a running session」メッセージが表示される

### 13-2. スレッド外（DM直接）からの送信
- [ ] スレッドではなくDMトップレベルにメッセージを送信
- [ ] 新しいスレッドとしてセッションが開始される（アンカー付き）

### 13-3. ボットメッセージの無視
- [ ] ボットの応答に対してリアクションを付ける
- [ ] ボットが反応しない（無限ループしない）

### 13-4. 空メッセージ
- [ ] 空のメッセージ（スペースのみ等）を送信
- [ ] 無視される

### 13-5. レート制限
- [ ] 短時間に10回以上メッセージを送信
- [ ] 「Rate limited. Please wait Ns.」が表示される
